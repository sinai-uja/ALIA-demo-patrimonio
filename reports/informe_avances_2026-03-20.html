🔍 Avance 2 — Búsqueda Facetada (Search)

Nuevo bounded context Search con búsqueda semántica, entity detection, filtros facetados, paginación y enriquecimiento con heritage assets.

Pipeline de búsqueda

1

Entity Detection

Detecta provincias, municipios y tipos de patrimonio en la query via regex accent-insensitive. Mapeo de keywords: “iglesia” → patrimonio_inmueble.

2a

Búsqueda Vectorial

pgvector cosine, top 200 candidatos.

2b

Búsqueda Full-Text

tsvector español, top 200 candidatos.

3

RRF Fusion + Filtro Relevancia + Reranking

Umbral 0.55. Pesos: 60% base 20% título 15% cobertura 5% posición

4

Agrupación + Asset Enrichment + Paginación

Agrupa por document_id, enriquece con heritage_assets (denominación, descripción, coordenadas, imagen, protección). Imágenes via cache IAPH.

🗺️ Avance 3 — Reestructuración Pipeline de Rutas

Pipeline rediseñado con enfoque stops-first, generación JSON estructurada por parada, enriquecimiento con heritage assets y guía interactivo mejorado.

Pipeline de 12 pasos

1

Limpieza + Extracción de query

Limpia texto, LLM extrae query concisa (máx 10 palabras).

2

RAG + Selección diversa de paradas

Recupera chunks (num_stops × 3). Round-robin por tipo de patrimonio para diversidad.

3

Heritage Asset Enrichment

Enriquece paradas con imágenes y coordenadas via HeritageAssetLookupPort.

4

Generación JSON Estructurada

LLM genera JSON con title, introduction, stops[].narrative, conclusion.

5

Composición + Persistencia

Compone narrativa monolítica, construye VirtualRoute con asset_id, image_url, lat/lon. Persiste en BD.

Ejemplo de salida JSON del LLM

{
  "title": "Ruta del patrimonio arqueológico jiennense",
  "introduction": "Un recorrido por los principales yacimientos...",
  "stops": [
    {"order": 1, "narrative": "El Castillo de Baños de la Encina..."},
    {"order": 2, "narrative": "En la cima del cerro de Santa Catalina..."}
  ],
  "conclusion": "Esta ruta ofrece una visión completa..."
}

Cambios clave

  • Paradas seleccionadas ANTES de la narrativa (stops-first)
  • JSON estructurado por parada con order y narrative
  • Imágenes y coordenadas desde heritage_assets
  • Columnas introduction / conclusion en BD
  • Guía con descripciones completas de assets (no chunks RAG)
