Excepción arquitectónica documentada (commit f4ec183)

El PgTraceRepository hace commit directo a la sesión en save_trace para no perder trazas cuando la transacción principal hace rollback. Un error en el pipeline RAG no debe impedir que la traza del fallo quede registrada. Se documenta explícitamente como excepción al patrón UnitOfWork.

⚡ Avance 4 — Robustez Cloud Run

Cuatro mejoras operativas para reforzar el despliegue Cloud Run tras la experiencia de la demo del 10 de abril (cold starts, falsos positivos de readiness).

Keepalive + status indicator (bbf2df5)

Endpoint GET /api/v1/health/services hace ping en paralelo a embedding-service y LLM-service. La NavBar muestra un indicador (verde/ámbar/rojo) para que el usuario vea si los servicios están calientes antes de lanzar una query. 636 líneas, 8 archivos.

Inference probes reales (54fe328)

vLLM responde 200 en /health antes de terminar de cargar pesos. Reemplazado por prueba real de inferencia (POST /v1/chat/completions con max_tokens=1) — única señal fiable. Identica para vLLM y llama.cpp.

Fix IAM audience (954f7ef)

Bug producción: token IAM pedido con audience https://uja-llm-xxx.run.app/v1 → Cloud Run 403. Fix: usar solo scheme://host como origin del audience. 

def _audience_for(target_url: str) -> str:
    parsed = urlparse(target_url)
    if parsed.scheme and parsed.netloc:
        return f"{parsed.scheme}://{parsed.netloc}"
    return target_url

Timeout vLLM 300 s (ad17ac8)

Cliente HTTP del backend eleva el timeout a 300 s para absorber cold starts largos de Cloud Run mientras vLLM carga pesos ALIA-40b.
